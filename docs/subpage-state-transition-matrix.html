Diff / Editor / History 子页面路由与状态记忆矩阵

本文档反映当前 diff 已落地的设计：三个子页面入口统一收口到 TerminalGrid.executeSubpageRoute，路由命令显式区分 open / switch / jump / close，每个页面通过 lifecycle 在离开前保存自己的状态。目标页面的实际回放仍由页面内部记忆完成，中心 SubpageStateMemory 当前承担统一 capture、scope 归一化和后续迁移基础的职责。

当前阶段：实现完成并已验证 统一 router lifecycle capture 中心 store 尚未统一回放 ProjectEditor GitDiffViewer GitHistoryViewer

1. 当前目标与验收口径

用户目标

在 Diff、Editor、History 三个子页面之间切换时，普通切换恢复目标页面上一次离开时的状态；来源页面的选中文件不能误覆盖目标页面自己的记忆。

已落地语义

switch 只表示页面切换；jump 和带文件目标的 open 才允许携带 filePath。因此 Diff 普通切到 Editor 与 Diff Jump to Editor 是两种不同路由。

最终验收

先用单测锁 route/memory 语义，再用功能 autotest 锁页面状态，最后用 CDP Input.dispatchMouseEvent 点击真实 UI 入口验收。

Markdown Preview 的问题已按“入口状态记忆”修复：Editor 经 Diff 或 History 返回时，保留 rendered HTML、preview scroll、编辑器可见性和 outline 目标，不再只支持快捷键关闭重开路径。

2. 当前实现拓扑

subpageRouter.ts 定义 SubpageRouteCommand、intent、entryPoint、file target。 shouldApplySubpageTargetFile() 保证只有 open/jump 应用文件目标。
TerminalGrid.tsx 唯一执行器：executeSubpageRoute()。 执行顺序：beforeLeave capture → set activeSubpage → open target → close non-target → afterEnter。
SubpageStateMemory 以 tabId + terminalId + root + subpage 归一化保存 snapshot。 当前用于统一 capture 和调试审计；页面恢复仍由各页面本地 memory 完成。
SubpagePanelShell 向父层发布 lifecycle。 保留 data-subpage-button，CDP 可用真实按钮坐标点击。
ProjectEditor 统一 soft-close snapshot，替代原先 retain-close 与 subpage-return 的双轨概念。 内部继续维护 ProjectEditorState.fileStates、Markdown session cache、PDF/EPUB reader memory。
GitDiffViewer / GitHistoryViewer Diff 在 beforeLeave 中保存 selected file、scroll、split ratio。 History 在 beforeLeave 中 flush selected commits/file 和三类 scroll。

3. 路由命令模型

字段 / intent 用途 当前入口 状态覆盖规则
open 从 Terminal / session / dropdown 打开某个子页面。 快捷键、下拉菜单、旧 open 事件、session restore。 如果携带 filePath，目标页面可应用文件目标；否则恢复目标页面自己的记忆。
switch 已在一个子页面内，普通切换到另一个子页面。 SubpageSwitcher、dropdown/shortcut 在已有子页面时触发。 永远不应用来源页面文件目标；目标页面恢复自己的 selected file / active file。
jump 显式深链：从当前上下文跳到目标文件。 Diff Jump to Editor、History Open Editor、Editor Jump/Back to Diff。 允许 filePath / repoRoot 覆盖目标页面当前选择。
close 从子页面返回 Terminal。 ESC、Return to Terminal 按钮、关闭事件。 先保存来源页面，再关闭所有非目标面板。
entryPoint 记录入口来源。 shortcut、dropdown、subpage-switcher、deep-link、escape、legacy-event。 用于调试、测试断言和后续迁移，不参与页面状态覆盖判断。

6. 状态记忆模型与文件类型边界

Scope Key
统一使用 { tabId?, terminalId, normalizedRoot, subpage }。Editor cwd、Diff cwd/repoRoot、History cwd/repoRoot 会先归一化，避免同一工作区产生多个状态桶。
beforeLeave
当前页面返回 snapshot：Editor active file + Markdown 状态摘要；Diff selected file + scroll + split ratio；History selected commits/file + scroll。router 将 snapshot 写入 SubpageStateMemory。
Route
router 根据 intent 决定是否应用 targetFile。普通 switch 不应用文件目标；jump 与带文件的 open 才应用。
afterEnter
目标页面挂载后执行页面自己的回放逻辑。Editor 的 Markdown cache hit 会保持 phase idle；Diff/History 继续从本地 memory/persisted state 恢复。
当前边界
中心 memory 已经统一保存各页面 snapshot，但尚未成为所有页面的唯一 replay 来源。后续若要彻底去掉页面内分散 memory，需要让 afterEnter 读取中心 store 并下发给目标页面。
文件 / 视图类型 当前会保持的状态 当前不会保持或尚无该视图 覆盖情况
Markdown Preview active file、Monaco viewState/cursor、preview 开关、editor 可见性、rendered HTML、preview scroll、outline target/scroll、file tree scroll。 无。 已由 PMSR + CDP 覆盖关键路径
普通文本 / 代码文件
包含 .html 源文件		 active file、Monaco viewState、cursor、outline scroll、file tree scroll、recent/pinned files。 没有 ProjectEditor 内置的“HTML Live Preview”面板；.html 当前作为 Monaco HTML 源码编辑。 基础位置记忆已覆盖
PDF active file、PDF page、PDF scrollTop、PDF scale、outline scroll。 PDF iframe 内部更细粒度临时 UI 状态不归 subpage router 管。 FileViewMemory 已支持
EPUB active file、EPUB CFI/location、scrollTop、font percentage、outline scroll。 epub.js iframe 内部临时状态不归 subpage router 管。 FileViewMemory 已支持
Image active file 和文件选择状态。 当前 image preview 没有 pan/zoom 控件，因此也没有 pan/zoom memory。 仅 active file 级别
SQLite active file 和打开状态。 SqliteViewer 的表选择、查询、滚动等内部 UI 状态没有接入 ProjectEditor fileStates。 需要单独扩展测试和 memory 字段
BrowserPanel / 外部网页预览 TerminalGrid 按 terminal 记住 last URL。 它不是 Editor/Diff/History 子页面，也不走本矩阵的 SubpageRouter memory。 另一个工具面板

9. 当前设计结论与后续补强点

普通切换语义

已确定：普通 SubpageSwitcher / dropdown / shortcut 切换恢复目标页面自己的上次状态，不携带来源页面 selected file。

深链语义

已确定：只有明确 Jump / Back / 外部打开指定文件，才允许覆盖目标页面 active file / selected file。

补强方向

若后续要继续削减页面内分散记忆，需要让 afterEnter 读取 SubpageStateMemory 并把 snapshot 显式下发给目标页面。

关于 HTML：当前 ProjectEditor 没有 Markdown Preview 同级的 HTML Preview 面板；.html 文件作为普通源码文件打开，Monaco 编辑器滚动、cursor、outline 和文件树位置会被保存。如果要新增真正的 HTML Preview，则需要为该 preview 定义独立的 scroll/DOM state memory，并补 CDP 点击与滚动恢复验收。
